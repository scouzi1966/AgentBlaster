# Evidence Bundles

Evidence bundle ZIPs package static, redaction-safe review artifacts for corporate approval, media support, or release governance. Campaign preflight folders are no-dispatch operator handoff artifacts; they can include local output, matrix, policy, and dry-run command paths and should be routed through release qualification bundles, evidence-index summaries, dashboard summaries, or explicit redaction review before external publication. They do not run benchmarks, contact providers, resolve secrets, execute tools, or read raw run artifacts.

## Create A Bundle

```bash
agentblaster evidence bundle \
  --suite-file examples/suites/toolsim.yaml \
  --policy agentblaster.policy.yaml \
  --include-provider-audit \
  --output-dir evidence \
  --audit-log audit/control-plane.jsonl
```

For a built-in suite:

```bash
agentblaster evidence bundle --suite trace-replay --output-dir evidence
```

## Create A Campaign Preflight Bundle

Campaign preflight bundles are folder artifacts for multi-matrix campaigns. They are designed for operator review before any provider dispatch:

```bash
agentblaster evidence campaign-preflight \
  --matrix examples/matrices/qwen-gemma-local.yaml \
  --matrix examples/matrices/qwen-gemma-stress.yaml \
  --policy agentblaster.policy.yaml \
  --benchmark-readiness reports/afm-trace-readiness.json \
  --output-dir campaign-preflight/qwen-gemma-local \
  --audit-log audit/control-plane.jsonl
```

For generated campaigns with many provider/model/suite readiness dossiers, prefer the list-file form generated by `agentblaster models campaign-plan`. Use one raw path per line; blank lines and lines starting with `#` are ignored, inline comments and shell-style quoted paths are rejected, and relative entries are resolved from the list file's directory:

```bash
agentblaster evidence campaign-preflight \
  --matrix campaigns/qwen-gemma-local/matrices/qwen-gemma-local.yaml \
  --policy agentblaster.policy.yaml \
  --benchmark-readiness-list campaigns/qwen-gemma-local/reports/benchmark-readiness-inputs.txt \
  --output-dir campaigns/qwen-gemma-local/reports/campaign-preflight
```

The command loads matrix and suite definitions, computes static case/token inventories, captures environment/package/implementation readiness, writes the artifact schema registry, optionally includes a redacted provider audit, optionally indexes benchmark-readiness dossiers supplied through repeated `--benchmark-readiness` flags or `--benchmark-readiness-list` files, and emits dry-run commands. Re-running it rewrites generated artifacts in place and removes stale optional readiness/provider audit artifacts that are no longer requested. It does not execute benchmark cases, contact providers, resolve API keys, read keyring values, or write run artifacts.
It also includes static matrix inventories and pressure audits so reviewers can see expected prompt, prefill, static-prefix, potential cache-reuse, output-token, and concurrency pressure before launching expensive local or remote runs.

## Contents

Each `.agentblaster-evidence.zip` contains fixed artifact names:

- `manifest.json`: bundle metadata, artifact SHA-256 checksums, and redaction/security notes.
- `suite-audit.json`: static suite provenance, risk, capability-surface, and dataset-hygiene audit.
- `harness-review.json`: static generated-suite governance review with provenance, surfaces, assertions, metrics, risks, and calibration requirements; prompts and provider payloads are excluded.
- `catalogs/simulated-tools.json`: deterministic simulated tool inventory.
- `catalogs/mcp-profiles.json`: deterministic MCP fixture profile inventory.
- `catalogs/lcp-profiles.json`: deterministic LCP-style context fixture inventory.
- `catalogs/skills.json`: bundled skill-pack inventory.
- `release-provenance.json`: project metadata, dependency declarations, and selected safe source hashes.
- `policy.yaml`: optional reviewed policy file when `--policy` is supplied.
- `provider-audit.json`: optional redacted provider inventory and policy audit when `--include-provider-audit` is supplied.

ZIP entries use fixed timestamps and sorted names to avoid unnecessary artifact churn. The bundle can be attached to benchmark reports or reviewed before running a paid remote matrix.

## Security Properties

Evidence bundles exclude provider configs, API keys, environment variables, keyring values, raw provider payloads, raw traces, run result rows, raw harness prompts in review artifacts, and dashboard state. When `--include-provider-audit` is used, the bundle includes only redacted provider audit metadata such as endpoint hosts, secret backend kind, writable/plaintext backend posture, TLS state, and policy findings. Including a policy file is intentional because policy is a reviewed governance artifact and should not contain raw secrets.

Evidence indexes summarize publication brief and SDLC validation manifest JSON artifacts without copying raw provider payloads, secrets, local paths, command output, or raw test logs. Publication brief summaries carry readiness and claim-evidence counts for corporate/media review, while SDLC validation manifest summaries carry test-tier, gate, Chrome GUI coverage, expected release evidence, and static security posture.

## Campaign Preflight Contents

Each campaign preflight folder contains fixed artifact groups:

- `manifest.json`: checksum index, matrix count, artifact count, optional `benchmark_readiness` index pointer, and explicit security posture.
- `RUNBOOK.md`: next no-dispatch dry-run commands and publication guidance.
- `readiness/environment-readiness.json`: static local runtime readiness.
- `readiness/implementation-status.json`: static implementation, requirement-coverage, harness-engineering suite, and stats-comparability inventory.
- `readiness/packaging-readiness.json`: static package metadata readiness.
- `readiness/benchmark-readiness-index.json`: optional compact benchmark-readiness summaries when `--benchmark-readiness` is supplied.
- `catalogs/artifact-schemas.json`: benchmark artifact schema registry.
- `matrices/index.json`: inventory index for each matrix.
- `matrices/*-inventory.json`: per-run suite, case, token, concurrency, raw-trace, model metadata, and compact suite-audit/dataset-hygiene inventory.
- `pressure/*-pressure.json`: per-matrix prompt, prefill, static-prefix, output-budget, and concurrency pressure audits.
- `policy/policy.yaml` and `policy/policy-normalized.json`: included only when `--policy` is supplied.
- `providers/provider-audit.json`: included unless `--no-provider-audit` is supplied.

The campaign preflight `readiness/implementation-status.json` artifact redacts local project-root and implementation file evidence paths. The raw `agentblaster implementation-status` report remains an internal readiness artifact because it can include local project-root and file evidence context.

`manifest.review_summary` is the compact no-local-path campaign preflight summary for evidence indexes and dashboard review. The rest of the raw preflight folder remains an operator handoff artifact because `manifest.json`, `matrices/index.json`, and `RUNBOOK.md` can include local output paths, matrix paths, policy paths, and dry-run commands.

Benchmark-readiness summaries preserve provider, suite, model, ready state, policy/suite compatibility, contract planning counts, metric coverage score, blocking/warning counts, and redacted provider-auth backend posture. They do not copy raw provider configs, API-key values, full secret reference names, prompts, traces, or endpoint payloads.

When the preflight manifest includes `benchmark_readiness.artifact_path`, `agentblaster release qualification-bundle --campaign-preflight-manifest campaign-preflight/<name>/manifest.json` archives a redacted compact copy, and `agentblaster release claim-readiness --campaign-preflight-manifest campaign-preflight/<name>/manifest.json` reads the compact index and carries the same readiness summaries into publication evidence.

Use campaign preflight bundles as the static handoff before expensive local Qwen/Gemma campaign runs or internet-facing OpenAI/Anthropic-compatible matrices.

## Create A Review Evidence Index

Evidence indexes are compact JSON artifacts for publication packets and release qualification workflows:

```bash
agentblaster evidence index \
  --name afm-release \
  --artifact reports/qwen-gemma-matrix-gate.json \
  --artifact reports/harness-orchestration-review.json \
  --artifact reports/afm-improvement-plan.json \
  --artifact reports/toolsim-suite-audit.json \
  --artifact reports/afm-metric-coverage.json \
  --artifact reports/manual-cleanup-plan.json \
  --artifact reports/cleanup-plan.json \
  --output-json reports/afm-release-evidence-index.json
```

The index records artifact status counts plus a `readiness` block with `ready`, `state`, blocking status names, review status names, and operator guidance. It also includes a top-level `cleanup_evidence` rollup with manual/retention report counts, plan/execution counts, audit-log-required coverage, and local-path/direct-publication-safety posture. It recognizes compact matrix-gate, implementation-status, campaign-preflight manifest, harness-review, engine-advisory, suite-audit, metric-coverage claim-contract posture, normalized-telemetry stats-comparability, benchmark-readiness auth-posture, provider-audit auth/security posture, provider-contract capability-evidence, cleanup plan/execution, retention cleanup, publication-brief, SDLC-validation, run publication-bundle readiness summaries, and matrix publication-bundle media-kit summaries without copying prompt text, raw findings, implementation file evidence paths, campaign matrix paths, campaign dry-run commands, cleanup path lists, raw telemetry usage/stats maps, telemetry source maps, per-field metric notes, provider payloads, API-key values, provider-audit finding messages, secret reference names, publication-brief proof text, SDLC fixture commands, raw test logs, or raw result rows. Artifact paths are reduced to safe names when absolute or parent-relative inputs are supplied, and entries record `path_redacted` to make that transformation explicit. Direct cleanup reports contain local path lists and are not publication-safe; the evidence-index cleanup summary is the shareable artifact. Release qualification bundles are summarized from `manifest.json` only, including embedded implementation-status, campaign-preflight, benchmark-readiness, provider-audit, normalized-telemetry, provider-contract, publication-brief, SDLC-validation, publication-bundle, and matrix-publication-bundle `review_summary` metadata when present. Run publication bundles are summarized from `publication-bundle-manifest.json` only, including media-kit roles/missing recommended assets, readiness status, and security booleans for raw secrets, raw provider payloads, and `results.jsonl`. Matrix publication bundles are summarized from `matrix-publication-bundle-manifest.json` only, including matrix artifact names, media-kit roles/missing recommended assets, and security booleans for raw secrets, provider payloads, `results.jsonl`, and per-run raw traces. `blocked` means failing or unreadable artifacts must be fixed before publication or release qualification. `review-required` means no blocking artifact statuses were found, but human signoff is still required for review-only or not-opened evidence. `ready` means all inspected artifact statuses are passing.
