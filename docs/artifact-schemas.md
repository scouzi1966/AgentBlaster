Provider audits are direct security review artifacts and also feed benchmark-readiness provider-auth posture without exposing raw secret names, paths, or values. Publication briefs and SDLC validation manifests are direct review artifacts and can also be carried through final archival release qualification bundles as compact summaries for claim-readiness, evidence-index, and dashboard consumers. Direct cleanup reports, raw implementation-status reports, broad readiness artifacts, and raw campaign preflight folders can contain local path context and are not directly publication-safe; use release qualification bundles, evidence-index summaries, dashboard review summaries, or explicit redaction review for publication or corporate review packets.

Inspect the static registry:

```bash
agentblaster catalog artifact-schemas --format markdown --output reports/artifact-schemas.md
agentblaster catalog artifact-schemas --format json --output reports/artifact-schemas.json
```

The registry is no-dispatch and no-inspection. It does not read run directories, provider configs, environment variables, keyrings, or remote endpoints.

Use the registry when:

- Building downstream report consumers.
- Reviewing which artifacts are publication-safe by design.
- Explaining why `results.jsonl`, `events.jsonl`, and `raw/` are not included in publication bundles by default.
- Creating corporate evidence checklists for benchmark campaigns.
